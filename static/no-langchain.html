What is LangChain?
The popular AI framework — and what it hides from you
LangChain is a Python framework that gives you pre-built components for building AI apps — agent loops, tool wrappers, LLM connectors, prompt templates, vector store integrations. You import it and wire things together. Most AI tutorials use it.
With LangChain
Import AgentExecutor, pass it tools, run it.

You don't see the prompt. You don't see how tools are called. You don't see how the LLM output is parsed. LangChain handles it all — invisibly.
Without LangChain (us)
Write the loop yourself. Write the prompt. Parse the LLM output with regex. Call APIs directly. Every step is visible, readable, and fully under your control.
# LangChain — 5 lines, but what's actually happening? from langchain.agents import initialize_agent, Tool from langchain.llms import OpenAI tools = [Tool(name="Search", func=search_fn, description="...")] agent = initialize_agent(tools, OpenAI(), agent="zero-shot-react-description") agent.run("What is Python 3.13?") # What happens inside? You have no idea unless you read LangChain source code. # How is the prompt built? How are tools parsed? How does it loop? Mystery.
LangChain is not bad — it is fast for prototyping. But when something breaks, you're debugging framework internals. When you want to change how the agent thinks, you're fighting abstractions. When you need custom streaming behaviour, you're writing workarounds.

Our Architecture — Every Layer Hand-Built
Six files. Zero AI framework dependencies.
🖥️
Frontend — index.html
Vanilla JS SSE reader, marked.js Markdown render, DOMPurify sanitiser, Chart.js charts, Google OAuth flow, sidebar history, char counter.

📦
API Server — main.py
FastAPI routes, Google OAuth via authlib, session middleware, input validation, Redis rate limiting, SSE StreamingResponse, SQLite history API.
main.py — 200 lines
🧠
ReAct Agent — agent.py
Hand-written ReAct loop (max 5 iterations). Direct httpx calls to Groq/Cerebras/Gemini/Ollama. call_groq_stream() for real-time token streaming. Regex-based output parser. Automatic provider fallback chain.
agent.py — 250 lines
🛠️
Tool Library — tools.py
11 tools written as plain Python functions. Tool dispatcher using re.match() to parse tool_name(argument) from LLM output. No framework schema required.
tools.py — 280 lines
⚡
Redis Cache — cache.py
Custom Redis manager with graceful no-op fallback. Search cache (1h TTL), URL cache (24h), rate limiter (INCR/EXPIRE), persistent user memory (no TTL).
cache.py — 120 lines
🗃️
Database — db.py
Raw SQLite3 — no ORM. Tables: users, sessions, messages. Functions: upsert_user, create_session, get_messages, add_message, auto_title, rename/delete session.
db.py — 120 lines
Total: ~1200 lines of Python + ~1200 lines of frontend. Every function has a clear purpose. Every file is readable. There is no magic import that does 10 things invisibly.

# How LangChain defines a tool: @tool def web_search(query: str) -> str: """Search the web for information.""" ... # How WE define a tool — identical function, no decorator needed: def web_search(query: str) -> str: cached = cache.get_search(query) if cached: return cached # ... scrape DuckDuckGo ... # Our dispatcher — 10 lines of regex, no framework: TOOLS = {"web_search": web_search, "code_exec": code_exec, ...} def dispatch_tool(action_str: str) -> str: match = re.match(r"(\w+)\((.+)\)", action_str) tool_name, arg = match.group(1), match.group(2) return TOOLS[tool_name](arg)

Why Build From Scratch?
The real benefits of not using a framework
✅ What we gained
🔍You can read every line of the agent — no mystery, no magic
🛠️True streaming was 40 lines — no callback workarounds
⚡Redis caching integrated exactly where we needed it
🔒Rate limiting, input validation — added in 10 lines
📦Fewer dependencies — faster installs, fewer version conflicts
🐛When something breaks, the bug is in your code — findable
🧠You actually understand how LLM agents work
❌ What we gave up
⏱️Took longer to build initially
📚No built-in document loaders (PDF, Word, etc.)
🔌No plug-and-play integrations (Slack, Notion, etc.)
🌐No community tool library to pull from
The bottom line: LangChain is excellent for rapid prototyping. But building from scratch meant every feature we added — Redis caching, true streaming, rate limiting, persistent memory — was exactly as we designed it, not a workaround against framework constraints. And more importantly: you now understand what an AI agent actually is.
What you actually learned by building this:

— How the ReAct (Reason + Act) pattern works at the code level
— How LLMs take a full prompt and predict the next token
— How SSE streams events from a server to a browser in real time
— How to parse free-form LLM output with regex
— How Redis caching, rate limiting, and persistent memory work
— How OAuth 2.0 login flow works end to end
— How to call Groq / Gemini / Cerebras APIs directly
— How to build a vector store RAG pipeline from scratch

If you had used LangChain, you would know how to configure it — not how any of this works.
