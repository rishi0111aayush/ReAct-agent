🤖 How Boozo.ai Works

Complete architecture — every layer from your question to the answer

v3 · Redis + Streaming + Markdown + 11 Tools

Step 1
You Type a Question
Open the chat at localhost:8080 and type your question. A live char counter tracks your input (max 4000 chars). You can also attach a PNG / JPG image for vision questions, or switch the AI model from the dropdown. Press Enter to send.
Enter to send Max 4 000 chars NEW: char counter Image upload Model switcher
🔐
Step 2
Google OAuth — Identity Check
The server checks who you are via Google OAuth 2.0. Logged-in users get per-user chat history (SQLite), session naming, and access to the remember / recall memory tools. Guests can still chat — history and persistent memory are just disabled.
authlib + itsdangerous Session cookie Guest mode
🛡️
Step 3
Input Validation + Rate Limiting
FastAPI checks that the message is not empty and under 4 000 characters — otherwise it returns a 400 error. Then Redis checks your request count: if you exceed 30 requests per minute, a 429 is returned. This protects API quotas from being drained.
400 if empty / too long Redis INCR + EXPIRE 30 req / min NEW: input limit
Redis
📦
Step 4
FastAPI /chat — SSE Stream Begins
Your message, conversation history, selected model, and user ID are packaged and passed to the ReAct agent. The response is a live SSE (Server-Sent Events) stream — events flow back in real time as the agent thinks, calls tools, and generates its answer.
POST /chat StreamingResponse SSE text/event-stream
🧠
Step 5
ReAct Agent Loop (up to 5 iterations)
The ReAct (Reason + Act) loop starts. Each iteration: build a prompt from your question + conversation history + scratchpad → call an LLM → parse the response → either dispatch a tool or give a Final Answer. The right panel shows thoughts, actions, and observations live.
Thought → Action → Observation Max 5 iterations Live scratchpad
⚡

Step 8
Answer Streams to Browser via SSE
Each event — thought, action, observation, answer_token, timing — arrives as a separate SSE message. The right panel updates live with the scratchpad, latency bars, model badge, and iteration dots. With Groq, the first word of the answer can appear in under 300ms.
✨
Step 9
Markdown Rendering + Syntax Highlighting
Once the done event arrives, the raw answer text is passed through marked.js for full Markdown rendering — bold, headers, lists, tables, blockquotes. Code blocks are syntax-highlighted by highlight.js. Output is sanitised by DOMPurify before being inserted into the DOM. Every code block gets a Copy button.

Step 10
Save History + Persistent Memory
The frontend calls /api/sessions/{id}/message to save both turns to SQLite. If the agent used remember(), the fact is written to Redis under your Google user ID — permanently, surviving restarts and new sessions. The RAG knowledge base is stored on disk via persistent ChromaDB (/data/chroma on Render).
SQLite — session history Redis — cross-session memory ChromaDB on disk — RAG

🗂️
Step 11
Session Auto-Title + Sidebar Refresh
After saving the first user message, the server runs auto_title() — it takes the first 40 characters of the message and sets it as the session name. The sidebar refreshes to show the updated title. You can also rename or delete sessions directly from the sidebar.
auto_title() PATCH /api/sessions/{id} DELETE /api/sessions/{id}
✅
Step 12
Done — Answer Displayed
The fully rendered Markdown answer sits in the chat. Charts render inline via Chart.js. History is saved. Switch to any old session from the sidebar to resume it exactly where you left off. The bot remembers everything you told it to remember — across all sessions, forever.

📚 Key Concepts

ReAct Loop — Reason + Act. Thought → Tool → Observation, up to 5 times.
True Streaming — Groq tokens arrive at the browser as the LLM generates them, not after.
Redis — Search cache (1 h), URL cache (24 h), rate limiting, permanent user memory.
RAG — Persistent ChromaDB on disk. Knowledge survives restarts.
SSE — Server-Sent Events. One HTTP connection, continuous event stream.
marked.js — Converts Markdown text to safe, sanitised HTML in the browser.
OAuth — Google login. User ID stored in a signed session cookie (itsdangerous).
remember / recall — Redis-backed tools. Facts persist across all sessions per user.
datetime_now — New tool. Gives the agent an accurate clock for any timezone.
wikipedia_search — New tool. Wikipedia REST API, no key needed.
get_weather — New tool. Live weather via wttr.in, no API key needed.
Fallback chain — Groq → Cerebras → Gemini → Ollama. Automatic, no user action.
