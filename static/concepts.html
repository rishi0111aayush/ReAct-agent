MCP — Model Context Protocol
The "plug socket" standard for connecting AI to tools

What is MCP? Imagine your AI assistant is like a smartphone. MCP is like the USB-C port — a standard connector that lets you plug in any accessory (charger, headphones, camera). MCP lets AI connect to any external tool, database, or data source using one standard language.

Where is MCP used in this project?

We added Context7 MCP — it sits between Claude Code (the AI coding assistant) and live library documentation. When Claude writes code for Boozo.ai, Context7 fetches the real, up-to-date docs for FastAPI, ChromaDB, Chart.js etc. so Claude doesn't use outdated API patterns.

ReAct Loop — The Agent's Brain
Reason + Act, repeated until the answer is ready

ReAct is the pattern Boozo.ai's brain follows. Instead of answering immediately, it thinks first, then acts, then thinks again — just like how you solve a hard exam question step by step rather than guessing instantly.

The exact loop (from agent.py):

1️⃣ Build the prompt — combine system instructions + conversation history + scratchpad
2️⃣ Call LLM — send the prompt to Groq/Cerebras/Gemini/Ollama
3️⃣ Parse output — find Thought / Action / Final Answer sections
4️⃣ If Action → run the tool, add result to scratchpad → go to step 1
5️⃣ If Final Answer → stream it to the user → stop
6️⃣ Max 5 iterations — stops automatically to prevent infinite loops

The scratchpad is like the agent's rough work book. Every Thought, Action, and Observation gets added to it. On the next loop, the LLM reads the full scratchpad so it knows everything it has already tried. You can see it live in the right panel of Boozo.ai!

SSE — Server-Sent Events
How the answer appears letter-by-letter in real time

Normal web request: You ask → server thinks → server sends the WHOLE answer at once → you see it.

SSE: You ask → server starts sending tiny pieces immediately → you see it building up in real time, like watching someone type.

How SSE works in this project (main.py):

The /chat endpoint returns a StreamingResponse. As the agent loop runs, each event (thought, action, observation, answer letter) is sent immediately as:

data: {"type":"thought","content":"I need to search..."}

The frontend JavaScript reads these one by one and updates the UI instantly. No waiting for the full answer!
📤 Events sent by server
iteration — which loop number
tokens — context window usage
thought — agent's thinking
action — tool being called
observation — tool result
chart — chart data
answer_token — one letter
done — final answer
error — something failed
📥 What frontend does
JavaScript listens to the SSE stream, reads each JSON event, and updates the right part of the UI:

• Brain steps panel
• Context window bar
• Tool usage bars
• Latency timeline
• Live scratchpad
• Chat bubble (letter by letter)
• Chart canvas (Chart.js)

RAG — Retrieval-Augmented Generation
Give the agent a memory / personal notebook

Problem: LLMs only know what they were trained on. They can't remember documents you give them in real time.

Solution: RAG gives the agent a searchable notebook. You store documents in it, and when you ask a question, relevant pieces are fetched and given to the LLM as context.

Tools — The Agent's Hands
Actions the agent can take in the real world

The LLM can only generate text — it can't browse the web or run code on its own. Tools are functions that bridge this gap. The agent writes Action: tool_name(argument) and the agent loop runs the real function and feeds the result back.

Tool File What it does in simple words Security
🔍 web_search tools.py Searches DuckDuckGo by scraping HTML. No API key needed. Read-only
💻 code_exec tools.py Runs Python code and returns the output printed to screen. AST sandbox — blocks attribute access so no escape
🧮 calculator tools.py Does maths safely. No sneaky code allowed — only +−×÷. Subprocess with 2s timeout — kills exponentiation bombs
📥 rag_ingest tools.py Stores a URL or text into the ChromaDB knowledge base. In-memory only
🔎 rag_search tools.py Finds the most relevant stored chunks for a question. Read-only
📊 plot tools.py Accepts a JSON chart spec and renders it with Chart.js in the chat. JSON validation only

LLM Providers & Fallback Chain
Four AI brains — automatic switch if one fails

Boozo.ai tries LLM providers in order. If the first one is busy, rate-limited, or broken, it automatically tries the next one — without you even noticing. It's like having four taxis on speed dial.

What is an LLM?
Large Language Model — a neural network trained on billions of text examples. It predicts the next word over and over to generate responses. GPT-4, Claude, Llama — all LLMs.
What is Ollama?
A tool that lets you run LLMs on your own computer (offline). Slower than cloud APIs but free forever and private — your data never leaves your machine.

Embeddings — Turning Words into Numbers
How computers understand meaning mathematically

Computers can't understand language — they only understand numbers. An embedding model converts any sentence into a list of ~384 numbers called a vector. Sentences with similar meaning get similar numbers — so "dog" and "puppy" end up numerically close, while "dog" and "aeroplane" are far apart.

Model used: all-MiniLM-L6-v2 from sentence-transformers. It's tiny (80MB) but fast and accurate enough for this project. When you call rag_ingest or rag_search, this model runs locally on your CPU to create or compare vectors. ChromaDB then finds the nearest vectors using cosine similarity.
Cosine similarity — simple version: Imagine two arrows pointing in space. If they point in almost the same direction → similar meaning. If they point opposite ways → completely different meaning. ChromaDB measures the angle between vectors to rank which stored chunks best match your query.

Security — The Code Sandbox
Making sure code_exec can't do evil things

Letting an AI run arbitrary Python code is dangerous. Without protection, someone could ask the agent to delete files, steal passwords, or crash the system. The sandbox prevents this with two layers.

🛡️ Layer 1 — AST Check
Before running, code is parsed into an Abstract Syntax Tree. If any node is an attribute access (x.y), it's rejected immediately. This blocks the classic Python escape: ().__class__.__bases__[0].__subclasses__()
🛡️ Layer 2 — Restricted Builtins
Only safe built-in functions are available: print, range, len, int, float, str, list, dict, sum, max, min, abs, round, sorted. No open(), no import, no exec().
Calculator extra protection: The calculator uses subprocess.run(..., timeout=2) — it runs in a completely separate OS process. If someone types 9**9**9**9 (which would hang Python forever), the OS kills the process after 2 seconds. The GIL (Python's lock) can't save you from this attack — only OS-level kill can.
